Technical Case Studies

LiquorCoordinator Route Optimization: A Technical Deep Dive

The Challenge

The Department of Liquor and Lottery's field coordinators needed a data-driven solution to efficiently plan their site visits. The primary goal was to identify agencies that were overdue for a visit (90+ days since the last inspection) and generate an optimized route for each coordinator that could be completed within a standard workday.

Data & Methodology

This project involved creating a robust data pipeline and a predictive model. The pipeline, built using Power Query (M Language) in Power BI, ingested and prepared data from multiple SharePoint sources, handling complex data transformations, column standardization, and data merging. The prepared data was then fed into a Python script.

The Python script performed two key functions:

  • **Geographic Clustering:** Using K-Means clustering from `scikit-learn`, the script grouped overdue agencies into geographically proximate clusters. The number of clusters was dynamically determined based on the total number of overdue agencies.
  • **Route Optimization:** Within each cluster, a nearest-neighbor algorithm was used to determine the most efficient visiting order. For travel time and distance, the script used the `openrouteservice` API, with a Haversine distance calculation as a reliable fallback.

Python Implementation Snippet

The following Python snippet demonstrates the core logic for the K-Means clustering and the subsequent route optimization using a nearest-neighbor approach. 


import pandas as pd
import numpy as np
from sklearn.cluster import KMeans
from sklearn.preprocessing import StandardScaler
import openrouteservice, os, warnings

os.environ['OMP_NUM_THREADS'] = '1'
warnings.filterwarnings('ignore')

# Note: The OpenRouteService API key has been removed for security.
client = openrouteservice.Client(
    key='YOUR_API_KEY_HERE'
)

def safe_numeric(s, default=0):
    """Converts a pandas Series to a numeric type, handling errors."""
    return pd.to_numeric(s, errors='coerce').fillna(default)

def optimize_overdue_visits(dataset: pd.DataFrame) -> pd.DataFrame:
    """
    Identifies overdue visits, clusters them geographically, and
    optimizes a visiting order for each cluster.
    """
    # Filter for agencies overdue by 90+ days
    overdue_raw = dataset[dataset['OverdueVisit90Days'] == True].copy()
    overdue_raw['SiteVisitDate_Clean'] = pd.to_datetime(overdue_raw['LastVisitDate'], errors='coerce')

    # Keep only the most recent overdue visit per agency
    overdue_sites = (
        overdue_raw.sort_values('SiteVisitDate_Clean', ascending=False)
        .drop_duplicates(subset='Agency', keep='first')
        .copy()
    )

    if overdue_sites.empty:
        return pd.DataFrame() # Return empty if no overdue sites

    # Feature scaling for K-Means clustering
    coords = overdue_sites[['Latitude', 'Longitude']].astype(float).values
    # Determine number of clusters, capped at 10
    k = max(2, min(len(overdue_sites) // 8, 10))
    coords_scaled = StandardScaler().fit_transform(coords)
    
    # Apply K-Means clustering
    overdue_sites['ClusterID'] = KMeans(n_clusters=k, random_state=42, n_init=10).fit_predict(coords_scaled)

    # Main routing loop to optimize each cluster
    routes = []
    for cid, grp in overdue_sites.groupby('ClusterID'):
        pts = grp.reset_index(drop=True)
        n = len(pts)
        order, visited, cur = [0], [True] + [False] * (n - 1), 0

        # Simple nearest-neighbor for route ordering
        while len(order) < n:
            lat1, lon1 = pts.at[cur, 'Latitude'], pts.at[cur, 'Longitude']
            dists = [
                np.hypot(lat1 - pts.at[j, 'Latitude'], lon1 - pts.at[j, 'Longitude'])
                if not visited[j] else np.inf
                for j in range(n)
            ]
            nxt = int(np.argmin(dists))
            order.append(nxt)
            visited[nxt], cur = True, nxt

        prev = None
        for seq, idx in enumerate(order):
            row = pts.iloc[idx].copy()
            row['VisitOrder'] = seq + 1

            if prev is None:
                row['TravelDistanceMiles'] = row['TravelTimeMinutes'] = 0
                row['DistanceMethod'] = 'N/A'
            else:
                # Use OpenRouteService API for realistic routing
                c_prev = (pts.at[prev, 'Longitude'], pts.at[prev, 'Latitude'])
                c_cur = (row['Longitude'], row['Latitude'])
                try:
                    res = client.directions([c_prev, c_cur], profile='driving-car')
                    summ = res['routes'][0]['summary']
                    row['TravelDistanceMiles'] = round(summ['distance'] / 1609.34, 2)
                    row['TravelTimeMinutes'] = round(summ['duration'] / 60, 1)
                    row['DistanceMethod'] = 'ORS'
                except:
                    # Fallback to Haversine if API fails
                    d = np.hypot(c_prev[0] - c_cur[0], c_prev[1] - c_cur[1]) * 69
                    row['TravelDistanceMiles'] = round(d, 2)
                    row['TravelTimeMinutes'] = round(d / 35 * 60, 1)
                    row['DistanceMethod'] = 'Haversine'
            
            routes.append(row)
            prev = idx

    return pd.DataFrame(routes)
