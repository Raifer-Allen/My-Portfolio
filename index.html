Technical Case Studies

From Theory to Implementation: A Technical Deep Dive

My projects start with a clear business problem, which I translate into a data-driven solution. This process ensures that every technical decision—from data modeling to algorithm choice—directly supports a tangible business outcome. The LiquorCoordinator Route Optimization project serves as a perfect example of this framework.

1. The Business Challenge

The Department's field coordinators needed an efficient way to plan their site visits, specifically to agencies that were overdue for an inspection (90+ days). The goal was to generate an optimized route that could be completed within a standard workday, minimizing travel time and maximizing efficiency.

2. Data & Theoretical Approach

The theoretical solution involved two key steps:

  • Geographic Clustering: The core idea was to group agencies that were close to each other. I chose a clustering algorithm, specifically K-Means clustering from `scikit-learn`, because it's a proven method for grouping data points based on their spatial coordinates.
  • Route Optimization: Once the agencies were grouped, the next theoretical step was to find the most efficient visiting order within each cluster. A nearest-neighbor algorithm was a practical choice, as it's a simple, effective heuristic for solving the Traveling Salesperson Problem (TSP) on a small scale.

3. Implementation & Iteration

The implementation began with building a robust data pipeline in Power BI to ingest and clean the data. The cleaned data was then passed to a Python script that put the theory into practice. This is where I introduced a critical layer of resiliency.

  • The script first applies K-Means clustering to the latitude and longitude of the overdue sites.
  • Then, a loop iterates through each cluster, using a nearest-neighbor approach to determine the visit order.
  • For real-world travel times and distances, the script attempts to use the `openrouteservice` API. However, because external APIs can sometimes fail, I built in a **critical fallback**: if the API call fails for any reason, the script seamlessly switches to a Haversine distance calculation. This ensures the process never breaks, providing a reliable and repeatable solution for the end-user.

Python Implementation Snippet

This snippet demonstrates the core logic for the K-Means clustering and the subsequent route optimization with the Haversine fallback. 


import pandas as pd
import numpy as np
from sklearn.cluster import KMeans
from sklearn.preprocessing import StandardScaler
import openrouteservice, os, warnings

# Ignore a specific warning from scikit-learn
os.environ['OMP_NUM_THREADS'] = '1'
warnings.filterwarnings('ignore')

# Note: The OpenRouteService API key has been removed for security.
client = openrouteservice.Client(
    key='YOUR_API_KEY_HERE'
)

def safe_numeric(s, default=0):
    """Converts a pandas Series to a numeric type, handling errors."""
    return pd.to_numeric(s, errors='coerce').fillna(default)

def optimize_overdue_visits(dataset: pd.DataFrame) -> pd.DataFrame:
    """
    Identifies overdue visits, clusters them geographically, and
    optimizes a visiting order for each cluster.
    """
    # Filter for agencies overdue by 90+ days
    overdue_raw = dataset[dataset['OverdueVisit90Days'] == True].copy()
    overdue_raw['SiteVisitDate_Clean'] = pd.to_datetime(overdue_raw['LastVisitDate'], errors='coerce')

    # Keep only the most recent overdue visit per agency
    overdue_sites = (
        overdue_raw.sort_values('SiteVisitDate_Clean', ascending=False)
        .drop_duplicates(subset='Agency', keep='first')
        .copy()
    )

    if overdue_sites.empty:
        return pd.DataFrame() # Return empty if no overdue sites

    # Feature scaling for K-Means clustering
    coords = overdue_sites[['Latitude', 'Longitude']].astype(float).values
    # Determine number of clusters, capped at 10
    k = max(2, min(len(overdue_sites) // 8, 10))
    coords_scaled = StandardScaler().fit_transform(coords)
    
    # Apply K-Means clustering
    overdue_sites['ClusterID'] = KMeans(n_clusters=k, random_state=42, n_init=10).fit_predict(coords_scaled)

    # Main routing loop to optimize each cluster
    routes = []
    for cid, grp in overdue_sites.groupby('ClusterID'):
        pts = grp.reset_index(drop=True)
        n = len(pts)
        order, visited, cur = [0], [True] + [False] * (n - 1), 0

        # Simple nearest-neighbor for route ordering
        while len(order) < n:
            lat1, lon1 = pts.at[cur, 'Latitude'], pts.at[cur, 'Longitude']
            dists = [
                np.hypot(lat1 - pts.at[j, 'Latitude'], lon1 - pts.at[j, 'Longitude'])
                if not visited[j] else np.inf
            ]
            nxt = int(np.argmin(dists))
            order.append(nxt)
            visited[nxt], cur = True, nxt

        prev = None
        for seq, idx in enumerate(order):
            row = pts.iloc[idx].copy()
            row['VisitOrder'] = seq + 1

            if prev is None:
                row['TravelDistanceMiles'] = row['TravelTimeMinutes'] = 0
                row['DistanceMethod'] = 'N/A'
            else:
                # Use OpenRouteService API for realistic routing
                c_prev = (pts.at[prev, 'Longitude'], pts.at[prev, 'Latitude'])
                c_cur = (row['Longitude'], row['Latitude'])
                try:
                    res = client.directions([c_prev, c_cur], profile='driving-car')
                    summ = res['routes'][0]['summary']
                    row['TravelDistanceMiles'] = round(summ['distance'] / 1609.34, 2)
                    row['TravelTimeMinutes'] = round(summ['duration'] / 60, 1)
                    row['DistanceMethod'] = 'ORS'
                except:
                    # Fallback to Haversine if API fails
                    d = np.hypot(c_prev[0] - c_cur[0], c_prev[1] - c_cur[1]) * 69
                    row['TravelDistanceMiles'] = round(d, 2)
                    row['TravelTimeMinutes'] = round(d / 35 * 60, 1)
                    row['DistanceMethod'] = 'Haversine'
            
            routes.append(row)
            prev = idx

    return pd.DataFrame(routes)
